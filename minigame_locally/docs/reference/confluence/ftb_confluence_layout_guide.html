Zona 5: Footer (layout completo)

Open Questions, VERIFY flags, y Related Docs. Esto puede quedar como está, al final de la página.

Cómo hacerlo en Confluence (paso a paso)

  1. Eliminar la tabla de "tabs"
    Selecciona toda la tabla (Game | How it works | Difficulty...) y bórrala. El Table of Contents macro ya hace de navegación — y funciona de verdad (lleva al anchor).
  2. Insertar un layout 30/70 debajo del ToC
    Pon el cursor debajo del Table of Contents → toolbar superior → icono de layouts (parece columnas) → elige "Sidebar izquierda" (30/70).
    En la columna izquierda: crea un {panel:title=Quick Facts} con los datos clave.
    En la columna derecha: mueve el Info panel de "What is FTB?" y la sección "How It Plays".
  3. Insertar un layout de 3 columnas para Design Dimensions
    Debajo del layout 30/70, inserta otro layout → "3 columnas iguales".
    Columna 1: Difficulty Model. Columna 2: Progression. Columna 3: Procedural.
    Cada una con un {panel} macro para darle borde visual.
  4. Envolver secciones largas con Expand
    Para cada sección larga (Curves, Progressions, Editor, Toolkit, Reporting): selecciona el contenido → /expand → ponle un título descriptivo.
    Esto colapsa el contenido por defecto — el lector solo lo abre si lo necesita.
  5. Convertir los "anchors" de la tabla de tabs en headings reales
    Si tenías anchors manuales para la tabla de tabs, conviértelos en h2 headings normales. El ToC los detecta automáticamente.

Macros que complementan los layouts

MacroShortcutPara qué
{expand}/expandColapsar secciones largas. El lector abre solo lo que necesita.
{panel}/panelCaja con borde y título. Ideal dentro de columnas para dar estructura visual.
{info}/infoCaja azul para contexto importante. Ya lo usas bien.
{note}/noteCaja amarilla para advertencias suaves. Ya lo usas bien.
{warning}/warningCaja roja para VERIFY flags y cosas que bloquean.
{status}/statusBadges de color (GREEN/YELLOW/RED). Útil en la sidebar para mostrar el estado de cada sección.
{toc}/tocTable of Contents automático. Ya lo tienes — mantenerlo.
{section} + {column}/sectionAlternativa a layouts — más control pero más manual. Solo si los layouts built-in no alcanzan.
